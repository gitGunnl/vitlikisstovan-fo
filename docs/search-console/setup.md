# Google Search Console + GA4 setup

Google Search Console (GSC) shows the Google search queries, impressions, and
rankings that bring people to the site — data that Google Analytics alone does
not have. This is especially useful here because of the many prerendered SEO
pages. Linking GSC to GA4 surfaces those search queries inside GA4's reports.

The site code already supports the **HTML-tag** verification method. Everything
else (completing verification, submitting the sitemap, linking to GA4) happens
in Google's dashboards and needs the owner's Google account.

## What's already wired in code

`client/index.html` contains:

```html
<meta name="google-site-verification" content="%VITE_GOOGLE_SITE_VERIFICATION%" />
```

At build time Vite replaces `%VITE_GOOGLE_SITE_VERIFICATION%` with the value of
the `VITE_GOOGLE_SITE_VERIFICATION` secret. Once that secret is set and the site
is redeployed, the deployed HTML carries your real verification token. Until
then the placeholder is inert (it does no harm and causes no SEO penalty — it
just won't verify yet).

## Owner steps (require a Google account)

### 1. Add the property and verify

1. Go to https://search.google.com/search-console and sign in.
2. Click **Add property** → choose **URL prefix** → enter
   `https://vitlikisstovan.fo`.
3. Pick a verification method:
   - **HTML tag (recommended, already wired):** Search Console shows a tag like
     `<meta name="google-site-verification" content="abc123..." />`. Copy only
     the token (the `content` value, e.g. `abc123...`).
     - In Replit, open the **Secrets** tab and add a secret named
       `VITE_GOOGLE_SITE_VERIFICATION` with that token as the value.
     - **Republish/redeploy** the site so the new build includes the token.
     - Back in Search Console, click **Verify**.
   - **DNS (alternative, no redeploy):** add the TXT record Search Console gives
     you to the `vitlikisstovan.fo` DNS zone, then click **Verify**. Use this if
     you prefer not to set the secret.
   - **HTML file (alternative):** download the `google<hash>.html` file from
     Search Console and place it in `client/public/` so it is served at the site
     root, then redeploy and **Verify**.

### 2. Submit the sitemap

1. In Search Console, open **Sitemaps** (left sidebar).
2. Enter `sitemap.xml` and click **Submit**.
   (Full URL: `https://vitlikisstovan.fo/sitemap.xml` — it is generated by the
   build and already served.)

### 3. Link Search Console to GA4

1. Open GA4 → **Admin** (bottom-left gear).
2. Under **Product links**, click **Search Console links** → **Link**.
3. Choose the verified `vitlikisstovan.fo` Search Console property, pick the web
   data stream, and confirm.
4. To see the data in GA4 reports: **Reports → Library**, then publish the
   **Search Console** report collection (GA4 hides it until published).

## Notes

- Verification method used by this repo: **HTML meta tag**, driven by the
  `VITE_GOOGLE_SITE_VERIFICATION` build-time env var (mirrors how
  `VITE_GA_MEASUREMENT_ID` configures GA4).
- The sitemap and SEO prerender logic were not changed for this task — only the
  verification tag was added.
